# 项目使用手册

## 1. 手册说明
本文档用于指导项目部署与日常使用，适用角色：系统管理员、资产专员、财务专员、普通员工。

## 2. 角色与权限
| 角色 | 主要权限 |
|---|---|
| 系统管理员 | 用户管理、部门管理、全模块访问 |
| 资产专员 | 资产管理、折旧规则、拍卖创建、处置确认、统计查询 |
| 财务专员 | 资产定价审核、付款审核、财务统计 |
| 普通员工 | 参与竞拍、查看我的竞拍/交易、确认成交、个人中心 |

## 3. 部署与启动
### 3.1 环境要求
- JDK 17+
- Maven 3.8+
- Node.js 18+
- MySQL 8.0+

### 3.2 数据库初始化
1. 创建数据库：`waidp_springboot_master`
2. 执行 SQL：`waidp_springboot_master.sql`

### 3.3 配置后端
编辑 `backend/src/main/resources/application.yml`：
- 数据库连接信息（URL/用户名/密码）
- `upload.path`（上传目录）
- 服务端口（默认 `8084`）

### 3.4 启动系统
方式一：
```bat
start_project_with_check.bat
```

方式二：
```bash
cd backend && mvn spring-boot:run
cd frontend && npm install && npm run dev
```

默认访问：
- 前端：`http://localhost:5173`
- 后端：`http://localhost:8084`

## 4. 登录与账号
- 默认管理员：`admin / admin123`
- 新用户可在登录页注册，注册账号默认角色为普通员工（`NORMAL_USER`）

## 5. 业务操作指南
### 5.1 系统管理员
1. 进入“用户管理”创建或维护账号（资产专员/财务专员/普通员工）。
2. 进入“部门管理”维护部门信息。
3. 在“统计”模块查看系统整体情况。

### 5.2 资产专员流程
1. 资产录入：在“资产管理”新增资产并上传图片。
2. 规则配置：在“折旧规则”维护折旧年限与残值率。
3. 等待财务审核：资产状态为“待审核”。
4. 创建拍卖：财务审核通过后（资产为“待拍卖”）创建拍卖活动。
5. 处置确认：付款审核通过后，在“资产处置”上传处置凭证并确认处置完成。

### 5.3 普通员工流程
1. 在“拍卖管理”查看可参与拍卖。
2. 进入拍卖详情进行“出价”或“一键出价”。
3. 在“我的竞拍”查看竞拍结果。
4. 中标后在“我的竞拍”执行“确认成交”。

说明：
- 系统生成交易后，中标者需在确认时限内确认（默认 24 小时）。
- 确认后会生成付款时限（默认 48 小时）。

### 5.4 财务专员流程
1. 在“财务审核-待审核”审核资产定价（可设置起拍价/保留价）。
2. 在“财务审核-付款待审核”审核成交付款，上传凭证并通过或拒绝。
3. 在“财务审核-审核历史”查看历史记录。

### 5.5 统计与归档
- “统计总览”：关键经营指标
- “资产处置统计”：按资产维度统计
- “员工竞拍记录”：按人员统计
- “处置档案查询”：已处置资产及交易档案

## 6. 状态流转说明
### 6.1 资产状态（常见）
- 待审核 -> 待拍卖 -> 拍卖中 -> 待处置 -> 已处置

### 6.2 拍卖状态
- `not_started`（未开始）
- `in_progress`（进行中）
- `ended`（已结束）

### 6.3 交易状态（关键字段）
- 成交确认：`pending` / `confirmed` / `expired`
- 付款状态：`pending` / `approved` / `rejected`
- 处置状态：`PENDING` / `COMPLETED`

## 7. 定时任务说明
- 每 60 秒：自动检查拍卖开始/结束状态
- 每天 02:00：检查中标后超时未确认的交易

## 8. 常见问题
### 8.1 前端打不开或接口 404
检查：
- 后端是否启动在 `8084`
- 前端 `vite` 代理配置是否指向正确端口

### 8.2 登录失败
检查：
- 数据库连接是否正常
- 账号是否被禁用
- 是否使用默认管理员 `admin/admin123`

### 8.3 上传文件失败
检查：
- `upload.path` 目录是否存在并可写
- 操作系统路径分隔符是否与配置一致
- 上传文件大小是否超出限制（后端默认单文件 10MB）

## 9. 运维建议
- 生产环境请移除明文密码与本地绝对路径
- 将 JWT 密钥改为环境变量注入
- 为数据库与 `uploads` 目录建立定期备份策略
- 发布前先验证关键流程：资产审核 -> 拍卖 -> 成交确认 -> 付款审核 -> 处置归档
