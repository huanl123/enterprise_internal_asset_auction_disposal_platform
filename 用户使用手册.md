时允许撤价；进入最后 12 小时后出价不可撤回# 企业废旧资产内部拍卖与处置平台 用户使用手册

适用范围：本仓库 `enterprise_internal_asset_auction_disposal_platform`（Spring Boot + Vue3 + Element Plus）。

## 1. 系统简介

平台用于企业内部废旧资产的建档、定价审核、内部拍卖、成交确认、收款审核、实物处置与归档查询，支持按角色分权操作。

系统角色：

- 普通员工：浏览拍卖、参与竞拍、确认成交、查看个人竞拍/交易单记录
- 资产专员：折旧规则、资产建档维护、创建拍卖、处置确认与归档
- 财务专员：资产定价审核（起拍价/保留价）、交易单收款审核（凭证）
- 系统管理员：用户/角色/部门管理；并拥有资产专员与财务专员的业务权限

## 2. 运行与访问

### 2.1 环境要求

- JDK 17
- Maven 3.8+
- Node.js 18+（建议 18/20 LTS）
- MySQL 8.x

### 2.2 后端配置

后端配置文件默认不入库，需要从示例复制：

1. 复制 `backend/src/main/resources/application.yml.example` 为 `backend/src/main/resources/application.yml`
2. 修改数据库连接（`spring.datasource.*`）
3. 按需修改上传目录（`upload.path`、`file.upload-dir`）

默认后端端口：`8084`（以 `application.yml` 的 `server.port` 为准）。

### 2.3 启动方式

一键启动（Windows）：

- 双击或命令行运行 `start_project_with_check.bat`
- 前端默认端口 `5173`，后端默认端口 `8084`

手动启动：

```bash
cd backend
mvn spring-boot:run
```

```bash
cd frontend
npm install
npm run dev
```

访问地址：

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8084`

可选：通过环境变量切换端口

- `VITE_BACKEND_PORT`：前端代理到的后端端口
- `VITE_FRONTEND_PORT`：前端启动端口

### 2.4 默认账号

后端启动时会初始化系统管理员账号（若不存在）：

- 用户名：`admin`
- 密码：`admin123`

普通员工账号：在登录页自助注册。
资产专员/财务专员账号：由系统管理员创建并分配角色。

## 3. 菜单与页面（前端）

不同角色登录后看到的菜单不同，常见入口如下：

- 首页：系统概览
- 用户与权限：用户管理、部门管理（仅系统管理员）
- 资产管理：资产列表、新增资产、折旧规则（资产专员/系统管理员）
- 拍卖管理：拍卖活动、我的竞拍、我的交易单、创建拍卖（按角色显示）
- 财务审核：定价审核/收款审核（财务专员/系统管理员）
- 资产处置：处置确认/凭证上传（资产专员/系统管理员）
- 统计查询：统计总览、处置档案查询、处置统计、员工竞拍记录（按角色显示）

如果页面出现中文乱码：通常是前端源码文件编码不一致导致，不影响后端业务；建议统一以 UTF-8 打开/保存前端源码后重新构建。

## 4. 通用操作

### 4.1 登录/退出

1. 打开前端地址进入登录页
2. 输入账号密码登录
3. 右上角头像菜单可进入“个人中心”或“退出登录”

### 4.2 权限提示

如果访问页面时自动跳回首页或提示无权限：说明该菜单/路由受角色限制，需要管理员调整角色或启用账号。

## 5. 系统管理员（admin）操作指南

### 5.1 部门管理

用途：为用户注册/创建时选择所属部门，且拍卖可按部门范围开放。

1. 进入“用户与权限 → 部门管理”
2. 新增/编辑部门（部门名称、负责人等）
3. 部门名称建议保持唯一，便于统计汇总

### 5.2 用户管理（创建与启用/禁用）

1. 进入“用户与权限 → 用户管理”
2. 新增用户：填写工号/账号、姓名、联系方式、部门、角色
3. 为资产专员/财务专员分配对应角色
4. 启用/禁用：被禁用账号将无法登录

说明：系统管理员同时拥有资产专员与财务专员权限，可参与资产建档、定价审核、拍卖发布、收款审核、处置归档等业务操作。

## 6. 资产专员操作指南

### 6.1 折旧规则管理

1. 进入“资产管理 → 折旧规则”
2. 新增折旧规则：规则名称、预计使用年限、残值率、说明
3. 规则被资产绑定后：仅可停用，不建议删除（防止历史资产关联丢失）

### 6.2 资产建档与维护

1. 进入“资产管理 → 新增资产”
2. 填写：资产编号（唯一）、名称、规格描述、使用部门、购置日期、资产原值
3. 上传多张实物照片
4. 选择一条有效折旧规则并保存

保存后系统将自动进行定价评估，资产状态进入“待审核”，等待财务专员审核通过后才能进入拍卖。

### 6.3 重新触发定价（针对未参与拍卖资产）

适用：资产仍处于“待拍卖”或流拍后重新处置，需要重新计算价值并再次走审核。
在资产列表/详情中使用“重新定价/重新发起审核”等按钮（以页面按钮为准）。

### 6.4 创建拍卖活动

前置条件：资产状态为“待拍卖”。

1. 进入“拍卖管理 → 创建拍卖”
2. 选择待拍卖资产
3. 设置拍卖开始/截止时间（拍卖时长不低于 5 分钟）
4. 设置加价幅度金额
5. 填写拍卖说明/须知
6. 设置参与范围：全公司或指定部门
7. 发布拍卖

### 6.5 处置确认（实物交割）

前置条件：交易单收款审核通过。

1. 进入“资产处置”
2. 打开对应处置单/交易单详情
3. 与中标者完成实物交割后，点击“确认处置”
4. 上传处置凭证（交割凭证/签收证明等）
5. 确认后资产状态更新为“已处置”，并形成归档档案

## 7. 财务专员操作指南

### 7.1 定价审核（起拍价/保留价）

前置条件：资产状态为“待审核”（系统已自动评估并给出建议起拍价）。

1. 进入“财务审核”
2. 打开待审核资产
3. 可选择开启“保留价”设置
4. 可调整起拍价与保留价
5. 审核通过：资产状态进入“待拍卖”
6. 审核不通过：填写原因，资产返回待修改状态（以页面状态为准）

说明：

- 起拍价：系统默认按当前价值的 80% 计算
- 保留价：对竞拍者不可见，出价需达到或超过保留价才算竞拍成功；否则视为流拍

### 7.2 交易单收款审核（付款凭证）

前置条件：中标者已确认成交并在规定时间内完成付款/提交凭证（以页面为准）。

1. 进入“财务审核”，找到待处理交易单
2. 点击“确认收款”，上传付款凭证
3. 选择审核结果：
   - 通过：流程进入资产专员处置环节
   - 不通过：填写原因，系统取消交易，资产恢复为“待拍卖”，并触发违约处罚机制

## 8. 普通员工操作指南

### 8.1 浏览拍卖与查看详情

1. 进入“拍卖管理 → 拍卖活动”
2. 通过拍卖名称/状态筛选
3. 点击“查看/竞拍”进入拍卖详情，查看资产信息、规则说明、竞价记录、剩余时间

### 8.2 参与竞拍（出价规则）

出价方式：

- 一键出价：按加价幅度自动加价
- 自定义出价：必须大于“当前价 + 加价幅度”

撤价（撤回出价）：

- 仅“当前最高出价者”可以撤价（按钮显示为“撤价”）
- 距离拍卖结束时间大于 12 小时时允许撤价；进入最后 12 小时后出价不可撤回

时间规则：

- 结束前 5 分钟内出价：自动延长 5 分钟（防止秒杀）

限制规则：

- 每人可同时参与多个拍卖
- 同一资产仅允许一人中标

### 8.3 成交确认与后续付款

拍卖结束后：

1. 若你为中标者，需在 24 小时内“确认成交”
2. 确认成交后系统生成《资产交易单》，并给出付款截止时间（默认 48 小时内）
3. 在规定时间内完成付款并按页面要求提交凭证/信息

违约说明（按需求约束）：

- 逾期未确认成交或放弃成交：将触发违约处罚（例如 3 个月竞拍禁用）
- 逾期未付款/付款审核不通过：交易可能被取消，资产重新进入待拍卖，并触发处罚

### 8.4 我的竞拍 / 我的交易单

1. “拍卖管理 → 我的竞拍”：查看参与场次、出价记录、中标情况
2. “拍卖管理 → 我的交易单”：查看生成的交易单、付款状态、截止时间等

## 9. 统计查询与档案查询

入口：“统计查询”。

- 统计总览：已处置资产、成交金额、流拍率等（资产/财务/管理员）
- 资产处置档案查询：按资产编号/名称检索全生命周期记录（含交易单、凭证、处置记录）
- 资产处置统计：按部门汇总处置数量、总成交金额、流拍率（支持按月/季/年筛选）
- 员工竞拍记录：参与场次、中标次数、违约次数、成交资产明细（支持按月/季/年筛选）

## 10. 关键业务规则（便于核对）

- 资产建档后系统自动定价评估，状态进入“待审核”
- 财务审核通过后进入“待拍卖”，资产专员才能创建拍卖
- 拍卖结束：
  - 若最高价达到（或超过）保留价（未启用保留价则按起拍价规则）：产生中标者并生成待确认信息
  - 否则视为流拍：资产回到“待拍卖”，可进入下一次拍卖
- 中标者确认成交时限：24 小时
- 付款截止时间：默认 48 小时（以交易单展示为准）
- 结束前 5 分钟内出价：自动延时 5 分钟
- 最高出价撤价窗口：拍卖结束前 12 小时外可撤价，最后 12 小时内出价不可撤回

## 11. 常见问题

### 11.1 无法登录

- 账号被禁用：联系系统管理员启用
- 密码错误：由管理员重置或重新注册（普通员工）

### 11.2 前端能打开但接口 404/500

- 检查后端是否启动、端口是否一致
- 检查 `frontend/vite.config.js` 的代理端口（`VITE_BACKEND_PORT`）
- 检查 `backend/src/main/resources/application.yml` 的数据库连接是否正确

### 11.3 启动报错：数据库没有表

项目默认 `spring.jpa.hibernate.ddl-auto: none`，需要导入初始化 SQL；本地快速体验可临时改为 `update/create`，但不建议用于最终提交或生产环境。
